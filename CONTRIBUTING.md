# Contributing

Contributions are welcome if they make a skill more reusable, clearer, or safer.

## What Makes A Good Skill

- Clear trigger: when should an agent use it?
- Concrete workflow: what steps should the agent follow?
- Safety gates: what actions need human approval?
- Verification: how does the agent know the work is actually done?
- Portability: no private paths, credentials, customer names, or machine-specific assumptions.

## How To Submit

1. Fork the repo.
2. Add or edit one skill at a time.
3. Put the skill in the closest category under `skills/`.
4. Update `CATEGORIES.md`.
5. Open a pull request with a short explanation of the workflow the skill supports.

Do not submit API keys, tokens, private customer examples, private project names, or local machine paths.
